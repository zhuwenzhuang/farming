# Farming v2.2.19

[English](./v2.2.19.md)

本次发布改进 Farming Code 的初始引导体验，使 Project 成员关系成为持久状态，并强化 Agent 恢复与应用包更新的正确性。

## 改进

- Farming Code 空工作区新增克制的操作引导，可直接进入 History、New Agent、Search、分享、专注模式和侧边栏控制；紧凑布局只保留两个主要操作。
- 新增移动端 Usage activity 视图，提供稳定的 52 周概览、精确日期选择和完整的 24 小时柱状图。
- History 行主体直接执行主要的打开、继续或恢复操作，同时保留明确的备用操作按钮。
- 移动端抽屉改为不透明背景，避免 Terminal 内容透过导航层干扰阅读。
- CRT 空状态统一命令风格，以 `[N]` 和 `[H]` 显示快捷键；Token 图表按后端实际返回的时间窗口标注。

## 修复

- Agent 恢复与服务重启后继续保留自定义标题、Project 工作区身份、排序、置顶和恢复元数据。
- Project 成员关系由 Backend 统一持有，覆盖 Agent 创建、History 恢复、Files、恢复的 Project Session 和 Git Worktree。加入失败时会精确回滚 Runtime 与持久成员关系，且不会产生 Archive History。
- Agent 重排、移除或恢复时，Project Files 始终绑定 Workspace Root。
- 应用包安装移入经过输入校验的独立 Helper，让运行中的 Server 能在重启前安全完成替换。

## 验证

- 187 个 Backend 与源码契约测试
- TypeScript Typecheck、ESLint、生产依赖审计与 `/farming` Production Build
- 6 个 Chromium CI 分片，零失败、零重试
- 真实 Codex 发布链路：ACP Chat、外观切换、CRT、Terminal 与 Resize
- npm 包与标准 App Bundle 的安装、Daemon、Native PTY、输入输出和关闭 Smoke Test

## 升级

```bash
npm install --global farming-code@2.2.19 && farming daemon
```
