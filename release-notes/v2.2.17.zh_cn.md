# Farming v2.2.17

[English](./v2.2.17.md)

这个版本让 Agent 在升级与重连后的恢复更加稳固，让更多正在进行的工作可以在 Farming Code 中即时切换，并为两套公开皮肤加入可扩展、可追溯到原始会话的 Token 历史统计。

## 重点改进

- **稳定的 Provider 会话身份**：Farming 继续使用稳定的 `fsess_*` 记录保存产品元信息，同时独立解析 Codex 和 OpenCode 的 Provider 身份。即使 Provider ID 暂时尚未生成，标题、置顶、Project 归属、运行模式和 Resume 行为也可以跨服务重启与 Archive/Restore 保留。
- **更安全的 Codex 与 OpenCode 启动**：Codex Terminal 仍然立即可交互，只有在唯一 Rollout 得到证明后才把临时关联 ID 升级为正式 ID；OpenCode 则在启动前创建并绑定准确的 Provider ID。两条路径都不依赖 `--last`，也不按时间选择“最近会话”。
- **可靠的后台重连**：浏览器切到后台再返回时，通过权威 Agent 状态和 Terminal Checkpoint 恢复，不再长期停在过期的心跳警告上，也不会回放历史 Terminal 输出。
- **Code 与 CRT 的 Token 活动视图**：Usage Panel 与 CRT Billing 共享按天、按小时的 Provider Token 历史、缓存占比、活跃天数、峰值日期和准确的选中日期详情。

## Token 历史

- 用只依赖随包运行时的 Node/TypeScript Worker 替代 Python Usage Helper。Codex 累积计数器与 Fork 统计改编自 MIT 协议的 CodexBar Parser；Claude 归一化遵循 MIT 协议的 `cc-statistics` 语义，来源记录在 `THIRD_PARTY_NOTICES.md`。
- 引入不兼容但可丢弃的新 SQLite Cache 版本，旧 Usage Cache 会自动清除重建；Codex、Claude 与 OpenCode 的原始历史始终保持只读和权威。
- 使用持久化目录清单与有界 Source Audit，Warm Refresh 不再枚举或排序全部历史会话。发生变化的文件从经过校验的字节 Checkpoint 继续；Rewrite-and-grow、截断、Archive Move 以及 Codex Parent/Fork 边界都有明确恢复语义。
- OpenCode 导出达到 5,000 个 Session 上限时会明确标记为 Partial，不再把截断结果展示成完整统计。
- 增加发布产物级 Smoke，验证打包后的 Worker、SQLite Cache 与真实格式的合成 Codex Token Event。

## Agent 生命周期与工作区

- 最多保留最近使用的 20 个 Agent View，使活跃任务切换能够保留滚动位置、编辑器状态和 Terminal 展示，同时限制浏览器内存增长。
- 服务升级时根据 Farming Session Metadata 恢复仍在临时 ID 阶段的 Codex Terminal，包括置顶和首页可见状态，同时避免把临时 ID 写入 Provider Membership。
- 收紧 Chat/Terminal 与权限模式转换：Terminal 一旦接受过输入，在可恢复 Provider Session 得到证明前不能静默切换。
- 在 Archive、Restore、Fork、运行时替换与 Native PTY Host 恢复过程中保持准确的 Provider Session Mapping。

## 验证

- 单文件 CLI 打包及其中 Usage Worker/SQLite Smoke
