# Farming v2.2.15

[English](./v2.2.15.md)

这个版本显著提升 Farming Code 在 iPhone 和桌面窄窗口下的可靠性，同时完成结构化 Coding Agent Chat 到 ACP 的迁移，并加强 Terminal 的精确恢复。

## 重点改进

- **手机和电脑共用一套紧凑布局**：桌面窄窗口与 iPhone 现在使用同一套响应式工作区，不再维护单独的移动端行为。Chat、Terminal、Project Files、抽屉、弹窗和附件控件共享稳定的几何与触控尺寸。
- **原生、可靠的输入框**：Chat Composer 在所有宽度下都使用浏览器原生 Textarea。点击、输入、选区、粘贴、中文输入法、键盘引发的视口变化、多行增高、附件操作和严格单次提交都不再依赖移动端输入代理。
- **Codex Chat 统一走 ACP**：Codex 的结构化 Chat 现在只使用锁定版本的 ACP Adapter。Chat/Terminal 切换、运行中追加指令、登录与权限卡片、Provider 能力、图片和音频附件都归入同一套 ACP 生命周期。
- **精确的 Terminal 展示**：Terminal Checkpoint 恢复、Resize 重绘、多窗口重连和焦点切换共同维护一份权威屏幕。Resize Burst 会原子绘制，重新打开或切换视图时不再暴露陈旧或半成品状态。

## 修复

- 防止紧凑布局 Overlay 和延迟焦点恢复导致 Composer 或 Terminal 点击后无法输入。
- 保持 Terminal 输入直接且严格单次发送，同时避免焦点操作触发完整 State Payload。
- 改善 iPhone 上 Agent、Chat、Terminal 与 Project Files 之间的导航，包括当前 Agent 的文件操作以及长、短 Transcript 场景。
- 空间足够时让短 Chat 回合从顶部开始，同时为长对话保留阅读位置。
- 稳定 ACP Follow-up Steering、活动标签、登录状态、权限请求和附件错误处理。
- 接受 Safari 录制的 WAV 音频；当实时 Agent 不支持某种媒体时提供可读的降级提示。
- 当 Registry Metadata 不可用时可靠回退，并在重启前核对实际安装的目标版本。

## 其他改进

- 增加可编辑、可持久化的 Farming 实例名称，并同步到工作区、浏览器标题、Code 和 CRT。
- 优化 Project Files 编辑器 Header 与紧凑文件导航。
- 保持 CRT 与 Farming Code 共用同一套 Agent 生命周期，同时改善 CRT 输入与紧凑展示。
- 删除旧的实验性 Codex App Server Chat 实现及其重复 UI 生命周期。
- 将锁定版本的 `codex-acp` 扩展补丁纳入 npm 包，并让 Package Smoke Test 验证安装后补丁确实生效。
