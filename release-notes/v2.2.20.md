# Farming v2.2.20

[简体中文](./v2.2.20.zh_cn.md)

This patch release improves CRT token telemetry and smooths token-usage interactions.

## Fixed

- Uses Provider-reported token telemetry for the CRT top-bar `TOK/MIN` value instead of estimating activity from terminal output.
- Makes CRT Billing Live use a one-hour timeline with one-minute buckets, bounded 15-second refreshes, and a visible sample time.
- Gives DAYS and LIVE distinct selected and keyboard-focus states so only the active tab appears selected.
- Adds a cancellable 200 ms hover delay to the Usage activity calendar. Quickly moving across dates no longer triggers intermediate requests or loading flashes; click, keyboard, and mobile date selection remain immediate.
- Aligns CRT calendar arrow-key movement with the visible week grid and prevents Main Agent onboarding from interrupting restored non-Agent views.

## Internal

- Renames the shared transcript implementation from Codex-first to Agent-first terminology while keeping provider-specific adapters explicit.
- Removes the obsolete fixed session-engine description from CRT Settings.

## Verification

- 187 backend and source-contract tests
- TypeScript typecheck and ESLint
- Three focused Chromium flows covering Usage hover, CRT Provider telemetry, and Billing Live

## Upgrade

```bash
npm install --global farming-code@2.2.20 && farming daemon
```
