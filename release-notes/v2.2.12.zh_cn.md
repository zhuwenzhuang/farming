# Farming v2.2.12

[English](./v2.2.12.md)

这个版本重点提升 Farming Code 的 Project 导航、Workspace 选择、Chat 输入区、Project Files 和本地 Token 活动展示，并补全多项 Project / Codex 生命周期操作，增加可安装桌面应用引导，同时刷新公开产品导览。

## Highlights

- **有序置顶与完整 Project 操作**：可在 `...` 菜单中置顶或取消置顶 Project；多个置顶项按加入顺序排列。菜单同时补齐在 Host 文件管理器中显示、创建永久 Git Worktree、重命名、全部标为已读、归档聊天、从 Farming 移除，以及对符合条件的 Worktree 执行显式危险删除流程。
- **每个 Project 独立控制 Agent 可见性**：Project 行新增紧凑的眼睛控件，只隐藏或恢复该 Project 的 Agent 列表；Files 继续可用，Project 标题与操作按钮保持稳定对齐。
- **原地逐步展开 Workspace 目录树**：New Agent 的 Host 目录选择器改为按需原地展开，不再逐级替换当前层。选择深层 Workspace 时会保留父级和兄弟目录上下文，远程浏览器同样选择服务端路径。
- **更有信息量的本地 Token 活动**：Usage 可展开为更大的 24 小时和 52 周视图，展示缩写 Token 总量、整点标签、选中日期读数、最近七天与此前七天对比、峰值日，以及按 Agent 类型聚合的分时直方图。Token King 使用皇冠轮廓，超过 1B Token 的日期使用火焰轮廓，同时保留原有热度颜色。
- **Chat 与 Terminal 输入区控制一致**：两种 Runtime 都使用输入区边缘悬停出现的收起 / 恢复操作。收起只隐藏输入区，保留未发送草稿和当前 Transcript / Terminal；Chat 末尾间距和复制按钮也进一步收紧，让最新结果更靠近输入框。
- **更一致的 Project Files 与轻量编辑体验**：Files、Changes、Untracked、History 和目录树统一紧凑字体、行高与层级间距。Markdown 链接会保留返回来源 Agent 的路径；JavaScript / TypeScript 编辑启用语法诊断，但不会伪装成完整 Project Typecheck。
- **有边界的 History 与明确的 Codex 归档行为**：History 使用稳定分页，只在页边界继续加载更早的 Provider Session。归档实时 Codex Agent 时先完成 Farming 本地生命周期，再对稳定 Session 执行 Provider Archive；Provider 失败会显式报告，但不会重新唤起 Agent。
- **可安装桌面应用引导**：Farming Code 提供应用 / 全屏入口和 Chrome 独立窗口安装说明。安装后的启动地址只保留干净 Base Path，并复用现有鉴权 Cookie，不会把启动 Token 固化到 URL。
- **更新产品导览与响应式布局**：公开 Code / CRT 截图与当前界面保持一致，界面切换路径写得更明确；Transcript 字体和宽窄布局也减少了空白与跳动。
- **更确定的浏览器覆盖**：Project 操作、Terminal 链接、Context Menu、Files、Chat 连续性、Workspace 选择、Usage 和平台差异改为使用稳定语义或 Test ID 契约验证。
