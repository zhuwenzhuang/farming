# Farming v2.2.1

[English](./v2.2.1.md)

这是 `v2.2.0` 之后的稳定性和发布准备补丁版本，重点覆盖 Farming Code 的 terminal 路径、agent 生命周期清理、隐藏标签页资源占用、工作区选择，以及 release 流程本身。

相比 `v2.2.0`，本版本让 native PTY 恢复和 archive 行为更不容易留下不可见 session；当 agent 被 archive 时，主页面 membership 会同步清理；隐藏浏览器标签页会减少后台 WebSocket 和轮询工作；后续 GitHub Release 也开始要求仓库内的 release note 文件作为正文来源。

## Highlights

- **Terminal 生命周期清理更严格**：native PTY 恢复时会杀掉不可恢复的 shell session，避免它们留在 native host 里但页面上没有对应 agent 行。
- **Archive 和主页面 session membership 对齐**：archive 可恢复的 Codex / Claude agent 时，会从 `mainPageSessionKeys` 移除对应 provider session，避免重启后悄悄恢复用户已经移出 live page 的 agent。
- **浏览器后台工作减少**：隐藏的 Farming 标签页会关闭 WebSocket，停止心跳类轮询，暂停 context window 刷新，并暂停 parked terminal 的 resize observer。
- **Agent 切换时 terminal 重绘更干净**：脱离可见 pane 的 terminal 保留状态，但不继续做不必要的 resize 工作；不可见输出会标记 unread，而不是强行触发多余的可见渲染。
- **New Agent 工作区选择支持更多目录**：路径建议数量更高，列表在对话框内滚动，键盘选择也能跟随滚动。
- **Release 流程开始进入仓库管理**：release workflow 要求存在 `release-notes/vX.Y.Z.md`，GitHub Release 正文从该文件生成，不再使用泛化内联文案。
- **打包态 CLI 启动 smoke 更少误报**：daemon 启动会给 packaged server 冷启动解压和 token 文件创建更充足的等待时间。

## Fixes

- 修复 server 重启后 recovered shell session 在 native PTY host 里累积的问题。
- 修复 provider-backed agent archive 后仍留在主页面 membership、后续可能被自动恢复的问题。
- 修复前端 archive 后没有立即应用后端返回的 removed main-page provider session keys。
- 修复隐藏标签页仍持续 WebSocket 和周期 fetch 的问题。
- 修复 parked terminal 脱离可见 pane 后仍持续观察 resize 的问题。
- 修复 New Agent 工作区路径建议数量太少、目录很多时视觉上被截断的问题。
- 修复 active-turn agent 行已经有运行状态指示时仍显示冗余时间的问题。
- 修复从 terminal 推断出的标题带有前置状态符号噪音的问题。
- 修复 packaged CLI release smoke 在冷启动时偶发因为 daemon ready timeout 太短而失败的问题。

## Features and Improvements

- 新增 native PTY shell session 恢复清理测试覆盖。
- 新增 provider-backed agent 和临时 shell agent 的 archive/history 测试覆盖。
- 新增工作区路径建议滚动行为的聚焦 Playwright 检查。
- 在 `AGENTS.md` 和 `AGENTS.zh_cn.md` 增加公开版本发布前门禁，覆盖版本对齐、本地和远程 dogfood smoke、打包依赖检查、release note 和 GitHub 发布检查。
- GitHub Release workflow 改为缺少对应 release note 文件时直接失败。

## Change Map

- `backend/agent-manager.js`：native PTY 恢复清理，以及 archive 时移除 provider-session membership。
- `src/hooks/useWebSocket.ts`、`src/hooks/usePageVisibility.ts`、`src/App.tsx`、`src/components/CodeWorkspace.tsx`：按页面可见性控制 WebSocket 和轮询工作。
- `src/lib/terminal-session-pool.ts`、`src/lib/terminal-output.ts`：parked terminal resize 和不可见输出渲染处理。
- `backend/server.js`、`src/components/InputDialog.tsx`、`src/styles/main.css`：更大且可滚动的工作区路径建议。
- `.github/workflows/release.yml`、`release-notes/v2.2.1.md`：release note 成为 tag 对应源码树的一部分。
- `backend/farming-app-cli.js`：packaged daemon 启动等待时间可配置，默认值放宽到更适合 release smoke 的窗口。

## Notes

- 本版本 GitHub Release 正文来自 `release-notes/v2.2.1.md`。
- package 版本号、Git tag `v2.2.1` 和 release note 文件名必须保持一致。
- 不需要手动配置迁移。
- 现有 `~/.farming/.session-token` 和 `~/.farming/settings.json` 应在升级后继续复用。
- 本版本没有故意移除 production dependency、native PTY asset、runtime file 或 install script。
- 本版本 macOS CLI 产物仍是 ad-hoc 签名。根据本机安全工具不同，首次运行可能需要手动允许一次，之后 packaged smoke 或正常启动才会成功。Developer ID 签名和 notarization 是后续发布加固工作。
- 正式推 tag 前应通过快速源码检查、聚焦 Playwright、本机 Mac 浏览器 smoke、远程 Linux dogfood smoke 和构建产物 smoke。

## 安装 / 升级

在 GitHub Releases 下载对应平台产物：

| 平台 | 文件 |
| --- | --- |
| macOS Apple Silicon | `farming_2.2.1_darwin_arm64` |
| macOS Intel | `farming_2.2.1_darwin_amd64` |
| Linux x64 | `farming_2.2.1_linux_amd64` 或 `farming-2.2.1.tar.gz` |
| Linux ARM64 | `farming_2.2.1_linux_arm64` |

老 Linux 如果需要 bundle 内置的 Node modules 和 glibc runtime，优先使用 app bundle tarball：`farming-2.2.1.tar.gz`。

macOS 如果下载后的 CLI 首次运行被拦截，请先在系统安全提示/设置中允许一次，再重新执行命令。
