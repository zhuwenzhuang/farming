# Farming v2.2.10

[English](./v2.2.10.md)

这个版本新增独立的 Farming Net 部署门户，把基于 VS Code 图算法的 Git History 工作流带入 Project Files，让运行中 Agent 控制更忠实地跟随实际 Runtime，并继续加固 Runtime 边界、Farming CRT、应用更新、远端部署和发布打包。

## Highlights

- **Farming Net 部署门户**：运行一套独立、带 Token 鉴权的目录页，统一进入本机、远端、内网和隧道中的 Farming。已登记目标通过短时、目标绑定、一次性的 Ed25519 签名通行证换取自身正常 Cookie；门户不保存也不暴露目标 Token。
- **Project Files 内的 Git History**：查看带分支、Tag、Merge Parent、变更文件和 Working Copy 状态的有界提交图，再把 Commit 或本地修改以正确捕获的 Revision 打开到 Review。Remove Project 仍然只移除项目成员关系、不删除文件系统内容，并新增了清晰的移除图标。
- **明确的 Runtime 与 Workspace 边界**：浏览器状态使用带 Tag 的 Runtime Binding 和后端拥有的 Runtime Observation；Provider 特定规划收敛在 Provider Adapter；常规终端元数据通过窄化的单 Agent Patch 更新；Project Files 的持久 Workspace Identity 不再依赖 Live Agent 的加载与排序。
- **需确认的 Workspace 创建**：当请求的 Workspace 目录不存在时，New Agent 会展示准确路径，并在创建目录和启动 Agent 前要求用户显式确认。
- **忠实的运行中 Agent 控制**：Codex Terminal 的模型、思考强度、Fast 和 Ultra 控制以当前 CLI Footer 与能力目录为准，不再被全局启动默认值覆盖。支持的修改会在下一条 Composer 消息前完成；没有可靠 Live Profile Adapter 的 Runtime 不显示当前 Session 控件。
- **更连贯的 ACP 工作流**：结构化 Chat 的历史回放和实时更新保持同一条有序 Transcript，排队输入与子 Agent 交互更安全，失败和认证前后的可见结果不会丢失，Chat / Terminal 切换继续绑定同一个 Provider Session。
- **更完整的键盘优先 CRT**：Code 与 CRT 切换的是同一个 Live Agent；全局关闭、Kill 和 Runtime 切换遵循一致的键盘状态契约；Dashboard 卡片显示紧凑的 Terminal 或结构化 Chat 活动；已死亡或归档 Agent 不再占用 Live Grid。只有 Main Agent 运行时，Dashboard 会恢复居中且可操作的 `[N] New Agent` 空状态，不再让主屏保持空黑。
- **更安全的更新和远端部署**：更新设置采用紧凑、显式的操作流；源码部署会在报告成功前验证远端健康状态、写入一致的 Server Control Metadata，并可在旧 Linux 主机上通过配置好的 glibc 2.28 Runtime 启动。
- **完整的 Runtime 打包**：npm 包和独立 App Bundle 现在都包含 Server 依赖的共享 Browser Protocol。Package Contract、Bundle Verification、安装与启动 Smoke 会防止不完整产物被发布。
- **可靠的旧版 Linux 启动命令**：App Bundle 在通过 glibc 2.28 Loader 启动 Node.js 前会先解析其绝对路径，因此 `url`、`status` 和 Agent 操作等 CLI 命令不会再在旧主机上被误当成共享库名称。
- **快速且可证明的 Terminal 恢复**：Native PTY Host 现在用 Runtime Epoch 和精确输出索引为权威 headless-xterm 状态生成 Checkpoint。页面长时间隐藏后会一次安装当前屏幕，不再逐秒追赶旧 TUI 重绘；`/session-view` 不再被轮询，重复消息会被忽略，索引缺口或 Runtime 变化通过一次权威 Checkpoint 恢复。Transport Failure 使用 Backoff，而持续返回非法 Checkpoint 时会显式停止，不会无限重试。

## Verification

- 182 项 Backend 与源码契约测试
- TypeScript typecheck 与 ESLint
- Project 移除与仅 Main Agent 时 CRT 空状态的定向 Chromium Playwright 覆盖
- 真实 Codex 跨 Skin 发布门禁：同一 Provider Session 完成 Code Terminal、ACP Chat、CRT、MSG、模型切换与 Resize 恢复
- 确定性 Terminal 输入门禁，Key-to-PTY-output p95 实测 64 ms（上限 250 ms）
- `/farming` 生产构建
- npm 包 smoke：全局安装、Daemon 启动、HTTP Auth Status 检查和原生 `node-pty` 加载
- 独立 App Bundle 校验与启动 Smoke，包含共享 Browser Protocol Runtime 依赖检查
- 与 `farming-code@2.2.8` 的包内容比较：没有移除任何顶层打包区域，新增 Farming Net Backend 与 Frontend 文件均已进入包内
- 对待推送文本历史执行私有路径、内部链接、Token 和密钥材料扫描

## 兼容性说明

- macOS CLI 产物使用 ad-hoc 签名，尚未进行 Developer ID 签名或公证。根据本机安全设置，首次运行可能需要先显式手动允许，再重新执行 CLI。

## 升级

```bash
npm install --global farming-code@2.2.10 && farming daemon
```
