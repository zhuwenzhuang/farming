# Farming v2.2.2

[English](./v2.2.2.md)

这是 `v2.2.1` 之后的 Farming Code 体验打磨版本，重点覆盖更安全且更有辨识度的诗意启动 token、workspace/file 交互、二维码分享，以及 shell prompt 保留。

相比 `v2.2.1`，Farming 现在会从更大的分来源语料中生成中文、日本风格、印度/泰戈尔风格的 5-7-5 启动 token，并保留保守熵检查。本版本也包含编辑器状态修复、二维码分享弹层、工作区启动控制改进，以及 shell prompt 处理改进。

## Highlights

- **诗意启动 token 更安全，也更有来源辨识度**：中文、日本风格、印度/泰戈尔风格 token 现在使用带质量分的分来源词表、来源锚点、保守熵估算，并减少跨来源 marker 混入。
- **Token 质量可以批量评估**：`scripts/evaluate-poetic-tokens.js` 会采样 token，检查熵、5-7-5 形态、唯一率、来源 marker 覆盖和强意象重复。
- **Workspace 启动更顺手**：Farming Code 增加更完整的新 agent 启动菜单，并扩大 workspace/path 建议覆盖。
- **编辑器状态在常见文件操作后更稳定**：打开的编辑器、working copy、保存和导航状态保留更一致。
- **二维码分享更易用**：share ticket 可以通过 QR popover 展示，复制行为也更清楚。
- **Shell prompt 保留更谨慎**：shell busy 集成避免覆盖用户自己的交互式 shell prompt 设置。
- **打包版 Terminal 启动更安全**：打包版 CLI 现在用最小环境启动 native PTY host，启动终端时避免把父进程环境写进日志。

## Fixes

- 修复 `心花`、`心弦`、`爱人`、`诗歌` 等印度/泰戈尔风格 marker 混入中文或日本 token 开头的问题。
- 修复偏情绪类词被当成通用景物意象使用的问题。
- 修复旧的 source-specific token 模板在实验启用时降低熵并重新引入噪音词的问题；这批死代码已清理。
- 修复编辑器 working copy 在保存和 tab 导航中的状态保留问题。
- 修复 shell prompt 初始化，Farming 的 shell hook 会保留用户自己的 prompt 设置。
- 修复 macOS 打包版 CLI 的 native PTY host 启动问题，避免打包二进制把旧的 host-mode 参数误当成模块路径。

## Features and Improvements

- 新增中国、日本风格、印度/泰戈尔风格 token 的生成词表。
- 在 `scripts/generate-poem-wordlists.js` 中加入 source word 质量元信息和抽取启发式规则。
- 新增 `scripts/evaluate-poetic-tokens.js`，作为诗意 token 的发布/诊断质量门禁。
- 新增 QR share ticket 后端和 UI 覆盖。
- 新增聚焦的 workspace/file 交互和 path suggestion 测试。

## Change Map

- `backend/haiku-token.js`、`backend/data/poetic-word-sources/*`：分来源诗意 token 生成和熵估算。
- `scripts/generate-poem-wordlists.js`、`scripts/evaluate-poetic-tokens.js`：语料抽取和 token 质量评估。
- `backend/tests/test-haiku-token.js`、`backend/tests/test-auth-token-file.js`：token 格式、来源 marker 和熵覆盖。
- `src/components/code/ShareQrButton.tsx`、`backend/qr-share-tickets.js`：二维码 share ticket 流程。
- `src/components/files/*`、`src/lib/workspace-*`、`backend/workspace-file-service.js`：workspace 文件和编辑器状态改进。
- `backend/shell-busy-integration.js`、`backend/local-session-engine.js`：shell prompt 保留。
- `backend/native-pty-host-client.js`：打包版 native PTY host 启动加固。

## Notes

- 本版本 GitHub Release 正文来自 `release-notes/v2.2.2.md`。
- package 版本号、Git tag `v2.2.2` 和 release note 文件名必须保持一致。
- 不需要手动配置迁移。
- 现有 `~/.farming/.session-token` 和 `~/.farming/settings.json` 应在升级后继续复用。
- 本版本没有故意移除 production dependency、native PTY asset、runtime file 或 install script。
- 本版本 macOS CLI 产物仍是 ad-hoc 签名。根据本机安全设置不同，首次运行可能需要手动允许一次，之后 packaged smoke 或正常启动才会成功。Developer ID 签名和 notarization 是后续发布加固工作。

## 安装 / 升级

在 GitHub Releases 下载对应平台产物：

| 平台 | 文件 |
| --- | --- |
| macOS Apple Silicon | `farming_2.2.2_darwin_arm64` |
| macOS Intel | `farming_2.2.2_darwin_amd64` |
