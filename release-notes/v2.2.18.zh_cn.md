# Farming v2.2.18

[English](./v2.2.18.md)

这是一个补丁版本，修复 Farming 升级后并发恢复多个 Codex Terminal 的问题。

## 修复

- Native PTY 恢复期间，按 Agent Home 串行启动 Codex Terminal。Farming 会等待一个 Codex 进程进入可见的就绪状态，再启动下一个共享同一 Codex 状态库的进程。
- 避免多个 `codex resume` 并发初始化 SQLite 时出现 `database is locked`，从而导致恢复后的 Project 短暂出现后又停止。
- 不同 Codex Agent Home 仍可独立启动，Shell、ACP、本地调试引擎和其他 Provider 的启动也不会因此延迟。
- 新增 Native PTY Rotation 回归测试：同时恢复四个同 Home 的 Codex Session，并使用并发启动即失败的 Fixture 验证串行边界。

## 验证

- 185 个 Backend 与源码契约测试
- TypeScript Typecheck 与 ESLint
- `/farming` Production Build
- 同 Home 启动冲突下的 Native PTY Rotation 回归测试

## 升级

```bash
npm install --global farming-code@2.2.18 && farming daemon
```
