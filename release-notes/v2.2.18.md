# Farming v2.2.18

[简体中文](./v2.2.18.zh_cn.md)

This patch release fixes concurrent Codex Terminal recovery after a Farming upgrade.

## Fixed

- Serializes Codex Terminal startup per Agent Home during native PTY recovery. Farming now waits for each Codex process to reach a visible ready state before starting the next process that shares the same Codex state database.
- Prevents concurrent `codex resume` processes from failing with SQLite `database is locked` errors, which could leave restored Projects briefly visible and then stopped.
- Keeps different Codex Agent Homes independent, and does not delay shell, ACP, local-debug, or other provider startup paths.
- Adds a native PTY rotation regression test that restores four same-home Codex sessions through a fixture that fails on overlapping startup.

## Verification

- 185 backend and source-contract tests
- TypeScript typecheck and ESLint
- `/farming` production build
- Native PTY rotation regression with same-home startup contention

## Upgrade

```bash
npm install --global farming-code@2.2.18 && farming daemon
```
