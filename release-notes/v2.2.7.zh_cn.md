# Farming v2.2.7

[English](./v2.2.7.md)

此版本让 Farming Code 与 Farming CRT 的模型和运行时控制更加真实一致，修复 History 无法搜索较早 Provider Session 的问题，并增强打包版本的启动与 npm 更新稳定性。

## 重点改进

- **History 改为检索完整 Provider 历史**：Farming Code 对非空 History 查询调用有边界的后端 Session 搜索，不再只过滤首批已加载记录。因此无需手动翻完所有分页，也能找到较早的 Codex、Claude Code、OpenCode 和 Qoder Session。搜索结果继续遵循 Farming 的已认领 Session 与主页面过滤规则，按 Provider 身份去重合并，并保留最新元数据。
- **搜索框只保留一个清除按钮**：History 与全局 Search 使用带 searchbox 语义的文本输入，Chromium 不会再在 Farming 自带的无障碍清除按钮旁渲染第二个原生叉号。
- **实时模型矩阵控制**：Codex 和 Claude Session 以紧凑矩阵提供适配 Provider 的模型、推理强度与速度选项。ACP 会原子应用相关配置；原生 Codex Terminal 会在发送排队 Prompt 前应用受支持的 `/model` 与 `/fast` 选择。
- **更安全的运行时切换**：新的 Codex 与 Qoder Session 可在 Provider 历史尚未落盘前切入结构化 ACP；已经接收过 Terminal 输入的 Session 仍必须存在可恢复的 Provider 记录。切换失败时会保留或恢复原运行时，并返回可操作的错误。
- **更完整的 CRT 遥测与 Transcript**：选中日期的 Billing 总量同时展示精确值与紧凑的 `K`/`M`/`B` 值；旧版 Turn 投影不可用时，CRT 也能渲染规范化结构 Transcript 条目。
- **更稳健的打包启动与更新**：Daemon 启动增加短暂存活稳定性检查，早退时会清理陈旧状态。npm 更新 Helper 从稳定配置目录运行、清除继承的 Server 模式环境变量，并在必要时保留打包 Loader 调用方式。
