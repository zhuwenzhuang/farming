# Farming v2.2.5

[English](./v2.2.5.md)

这是 `v2.2.2` 之后的 Farming Code 体验打磨版本，重点覆盖项目文件导航稳定性、terminal 行为、agent 启动配置，以及公开 README。

相比 `v2.2.2`，Farming Code 现在会在文件树滚动或自动 reveal 时保留 Open Editors 可见；打开文件较多时，Open Editors 自己限制为 7 行并内部滚动。本版本也继续改进 terminal / shell 状态处理，并把 README 截图和功能说明更新到当前产品状态。

## Highlights

- **浏览 Files 时 Open Editors 不再丢失**：Open Editors 最多保持 7 行可见，超过后内部滚动，文件树 reveal 不会把它顶出可视区域。
- **文件导航更接近 VS Code**：文件树 reveal 会避开 project 行、agent 行、Open Editors、Files header 和 sticky folder context 后再滚动。
- **Project Files review 更容易扫**：侧边栏把 Changes、Open Editors、文件搜索、预览、diff、blame 放在同一个 project-scoped 流程里。
- **Terminal 和 shell 状态更稳**：shell 环境解析、terminal busy 状态、prompt 保留、active 判断和 terminal link 行为继续收敛。
- **Agent 启动控制更清楚**：Codex / Claude 只展示 Farming 能传给底层 CLI 的 provider-specific 选项。
- **README 回到当前状态**：替换主工作台截图，并在用户功能说明里补充 Open Editors、Changes、provider controls、预览能力和 usage/context signals。

## Fixes

- 修复选择或自动 reveal 深层文件时 Open Editors 消失的问题。
- 修复 Open Editors 文件较多时不做高度限制、导致挤占侧边栏的问题。
- 修复 file reveal 把 sticky 区域也当作可视区域，导致目标行被顶层区域遮住的问题。
- 修复若干 terminal 状态边界，包括 shell activity、prompt 保留和链接目标。
- 修复 workspace/file 交互细节，包括 changes、preview、diff、tabs 和键盘导航。
- 修复 README 和当前 Farming Code 界面不一致的问题。

## Features and Improvements

- 新增 Open Editors 7 行上限、内部滚动和 active editor 自动可见。
- 新增大 Open Editors 列表 + 深层文件 reveal 的 Playwright 覆盖。
- Project Files sticky context 现在把 Open Editors 纳入文件树可见边界。
- 合并 `v2.2.2` 之后的 Farming Code 桌面和移动端 shell 体验打磨。
- 更新 README / README.zh_cn 的用户功能摘要。
- 替换 README 主工作台截图。

## Notes

- 本版本 GitHub Release 正文来自 `release-notes/v2.2.5.md`。
- package 版本号、Git tag `v2.2.5` 和 release note 文件名必须保持一致。
- 不需要手动配置迁移。
- 现有 `~/.farming/.session-token` 和 `~/.farming/settings.json` 应在升级后继续复用。
- 本版本没有故意移除 production dependency、native PTY asset、runtime file 或 install script。
- 本版本 macOS CLI 产物仍是 ad-hoc 签名。根据本机安全设置不同，首次运行可能需要手动允许一次，之后 packaged smoke 或正常启动才会成功。Developer ID 签名和 notarization 是后续发布加固工作。

## 安装 / 升级

在 GitHub Releases 下载对应平台产物：

| 平台 | 文件 |
| --- | --- |
| macOS Apple Silicon | `farming_2.2.5_darwin_arm64` |
| macOS Intel | `farming_2.2.5_darwin_amd64` |
| Linux x64 | `farming_2.2.5_linux_amd64` 或 `farming-2.2.5.tar.gz` |
| Linux ARM64 | `farming_2.2.5_linux_arm64` |

老 Linux 如果需要 bundle 内置的 Node modules 和 glibc runtime，优先使用 app bundle tarball：`farming-2.2.5.tar.gz`。

macOS 如果下载后的 CLI 首次运行被拦截，请先在系统安全提示/设置中允许一次，再重新执行命令。
