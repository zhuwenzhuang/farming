# Farming v2.2.17

[简体中文](./v2.2.17.zh_cn.md)

This release makes Agent recovery more durable across upgrades and reconnects, keeps more active work instantly available in Farming Code, and adds scalable, source-backed token history to both public skins.

## Highlights

- **Durable provider-session identity**: Farming now keeps product metadata under its stable `fsess_*` record while resolving Codex and OpenCode provider identities independently. Titles, pins, Project membership, runtime mode, and resume behavior survive temporary provider IDs, server restarts, and archive/restore transitions.
- **Safer Codex and OpenCode startup**: Codex Terminal remains immediately interactive and promotes its temporary correlation ID only after a unique rollout is proven. OpenCode sessions are created with their exact provider ID before launch. Neither path relies on `--last` or a timing-only nearest-session guess.
- **Responsive reconnect recovery**: backgrounded browsers recover through the authoritative Agent and terminal checkpoint state instead of remaining stuck behind a stale heartbeat warning or replaying historical terminal output.
- **Token activity in Code and CRT**: the Usage panel and CRT Billing view now share daily and hourly provider-backed token history, cache composition, active days, peak days, and exact selected-day details.

## Token history

- Replaced the Python usage helper with a Node/TypeScript worker using only bundled runtime facilities. Codex cumulative-counter and fork accounting is adapted from the MIT-licensed CodexBar parser; Claude normalization follows the MIT-licensed `cc-statistics` semantics, with attribution recorded in `THIRD_PARTY_NOTICES.md`.
- Added an incompatible, disposable SQLite cache revision that automatically rebuilds old usage caches. The original Codex, Claude, and OpenCode histories remain read-only and authoritative.
- Added a persistent directory census and bounded source audit so warm refreshes do not enumerate or sort every historical session. Changed files resume from validated byte checkpoints; rewrite-and-grow, truncation, archive moves, and Codex parent/fork boundaries have explicit recovery behavior.
- Marked truncated OpenCode exports as partial instead of presenting their first 5,000 sessions as complete.
- Added release-artifact smoke coverage for the bundled worker, SQLite cache, and a real synthetic Codex token event.

## Agent lifecycle and workspace

- Kept up to 20 recently used Agent views mounted so switching among active tasks preserves scroll position, editor state, and Terminal presentation without unbounded browser growth.
- Restored temporary Codex Terminal records from their Farming session metadata during server upgrades, including pinned and main-page visibility, without adding temporary IDs to provider membership.
- Tightened Chat/Terminal and permission-mode transitions so a Terminal that has accepted input cannot silently switch before a resumable provider session is proven.
- Preserved exact provider-session mapping across archive, restore, fork, runtime replacement, and native PTY host recovery.

## Verification

- TypeScript typecheck, ESLint, backend/source-contract tests, and a `/farming` production build
- Focused Chromium coverage for Code and CRT token history, background reconnect, retained Agent views, and provider-session recovery
- Real Codex cross-skin and direct terminal-input release gates
- Standalone CLI packaging and bundled usage-worker/SQLite smoke
- Read-only Code and CRT validation against a long-running Linux environment with active Agents preserved
