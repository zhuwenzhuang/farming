# Farming v2.2.15

[简体中文](./v2.2.15.zh_cn.md)

This release makes Farming Code substantially more reliable on iPhone and narrow desktop windows, while completing the move to ACP for structured coding-agent Chat and strengthening exact terminal recovery.

## Highlights

- **One compact layout across phone and desktop**: narrow desktop windows and iPhone now use the same responsive workspace instead of separate mobile behavior. Chat, Terminal, Project Files, drawers, dialogs, and attachment controls share stable geometry and touch targets.
- **Native, dependable composer input**: the Chat composer now uses the browser's native textarea path at every width. Clicking, typing, selection, paste, Chinese IME, keyboard resize, multiline growth, attachment actions, and exactly-once submission remain available without a mobile-only input proxy.
- **Codex Chat through ACP**: Codex structured Chat now uses the pinned ACP adapter as its single supported path. Chat/Terminal switching, live steering, authentication and permission cards, provider capabilities, images, and audio attachments reduce through the shared ACP lifecycle.
- **Exact terminal presentation**: terminal checkpoint recovery, resize redraws, multi-view reconnects, and focus transitions preserve one authoritative screen. Resize bursts paint atomically and reopening or switching views no longer exposes stale or partial state.

## Fixed

- Prevented compact-layout overlays and delayed focus restoration from making the composer or Terminal impossible to type into.
- Opened the Terminal composer by default for first-time desktop users and kept its native textarea compact when empty while growing with multiline content.
- Kept Terminal input direct and exactly once while avoiding focus-triggered full-state payloads.
- Improved iPhone navigation between Agents, Chat, Terminal, and Project Files, including active-Agent file actions and long or short transcript states.
- Started short Chat turns at the top when there is enough room, while preserving reading position for long conversations.
- Stabilized ACP follow-up steering, activity labels, authentication states, permission prompts, and attachment error handling.
- Accepted Safari-recorded WAV audio and exposed readable fallbacks when the live Agent does not support a media type.
- Made update discovery fall back cleanly when registry metadata is unavailable and verify the installed target version before restart.

## Other improvements

- Added editable, persistent Farming instance names across the workspace, browser title, Code, and CRT.
- Refined the Project Files editor header and compact file navigation.
- Kept CRT and Farming Code connected to the same Agent lifecycle while improving CRT input and compact presentation.
- Removed the former experimental Codex app-server Chat implementation and its duplicate UI lifecycle.
- Included the pinned `codex-acp` extension patch in npm packages and made package smoke tests verify that it is applied after installation.
- Updated the Express, Hono, and DOMPurify dependency chain to current patched releases; the production dependency audit is clean.

## Verification

- TypeScript typecheck, ESLint, 182 backend/source-contract tests, and a `/farming` production build
- Focused Chromium coverage for iPhone layouts, mobile human workflows, terminal recovery, resize coordination, link targets, and direct input latency, with retries disabled
- Real-Agent iPhone screenshots across bash, Codex, and OpenCode, plus the real Codex cross-skin release case
- Global npm package installation, patched ACP verification, server startup, and native `node-pty` loading

## Upgrade

```bash
npm install --global farming-code@2.2.15 && farming daemon
```
