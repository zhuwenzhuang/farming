# Farming v2.2.8

[English](./v2.2.8.md)

这个补丁版本让结构化 Chat 在浏览器页面切到后台后仍然持续更新，同时加固 OpenCode 新会话的 ACP 启动，并带上主分支中尚未发布的模型控制和产品文档改进。

## Highlights

- **页面隐藏后 Chat 仍会继续**：Farming Code 不再因 `visibilitychange` 主动关闭共用的后端 WebSocket。ACP transcript revision 和终端输出在后台标签页中仍会持续到达，回到页面时也不会再制造一次假的“已断开 / 正在重连”状态。Usage、Context、Session ID 和相对时间等非关键轮询在隐藏时仍会暂停。
- **OpenCode 新会话保持 ACP Chat**：刚启动的 OpenCode Agent 不会再因为 Provider Session ID 尚未生成而退回 Terminal。Terminal 已收到输入后的可恢复 Session 保护仍然保留，已有对话不会被静默丢弃。
- **运行中模型控制更连贯**：紧凑模型矩阵和 Advanced 控件在高度与淡入淡出过渡中保持挂载并保留当前选择；普通模型或思考强度变化不会重复播放 Ultra 摇杆动画。激活 Ultra 时，充能轨道和拉杆会切换为克制的紫色荧光，并在自动下落到终点后产生一次短促震动。Terminal Codex 使用同一套能力感知矩阵，并在下一条 Prompt 前应用支持的运行中 Profile 修改。
- **产品指南和截图同步当前能力**：README、Farming Code 与 Farming CRT 指南更直接地说明两套界面、支持的 Agent 路径、Review、模型控制、手机使用方式和当前产品截图。

## Verification

- 145 项 Backend 与源码契约测试
- TypeScript typecheck 与 ESLint
- 页面隐藏时 ACP 连续更新、OpenCode 新会话 ACP 启动、ACP / Terminal 模型控制的定向 Playwright 覆盖
- 有界面 Chrome 页面冻结 / 恢复验证：ACP 任务完成、共用 WebSocket 保持连接，恢复后答案直接出现且没有重连提示
- `/farming` 生产构建和 npm 打包 smoke test
