# Farming v2.2.19

[简体中文](./v2.2.19.zh_cn.md)

This release improves the Farming Code getting-started experience, makes Project membership durable, and strengthens Agent resume and bundle-update correctness.

## Improved

- Adds a focused empty-workspace guide in Farming Code with direct actions for History, New Agent, Search, sharing, focus mode, and sidebar control. Compact layouts retain the two primary actions.
- Adds a mobile Usage activity view with a stable 52-week overview, exact day selection, and a complete 24-hour histogram.
- Makes History rows directly perform their primary open, continue, or resume action while preserving explicit alternate controls.
- Keeps mobile drawers opaque so terminal content does not bleed through navigation.
- Aligns the CRT empty state with its command style by displaying the `[N]` and `[H]` shortcuts consistently, and labels token charts from their actual reported time window.

## Fixed

- Persists renamed Agent titles, Project workspace identity, ordering, pinning, and recovery metadata across resume and restart boundaries.
- Makes Project membership backend-authoritative across Agent creation, History resume, Files, restored Project sessions, and Git worktrees. Failed admission now rolls back the exact runtime and durable membership without creating archive history.
- Keeps Project Files bound to the workspace root while Agents are reordered, removed, or resumed.
- Moves app-bundle installation into a validated detached helper so the running server can finish a replacement safely before restart.

## Verification

- 187 backend and source-contract tests
- TypeScript typecheck, ESLint, production dependency audit, and `/farming` production build
- Six Chromium CI shards with no failures or retries
- Real Codex release flow across ACP Chat, appearance switching, CRT, Terminal, and resize
- npm package and standard app-bundle installation, daemon, native PTY, input/output, and shutdown smoke tests

## Upgrade

```bash
npm install --global farming-code@2.2.19 && farming daemon
```
