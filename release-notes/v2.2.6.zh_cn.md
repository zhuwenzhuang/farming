# Farming v2.2.6

[English](./v2.2.6.md)

本版本把 Farming 从单一工作台继续推进为更完整的多 Agent 开发环境：强化 Farming Code 的 review / 文件工作流，在同一套后端 session 上增加独立的 Farming CRT 实时皮肤，扩展 Coding Agent runtime 支持，并加固 session 恢复、更新、打包和发布验证。

## Highlights

- **两套实时 UI，共用一套 runtime**：Farming Code 仍是 `/code/` 默认界面，`/crt/` 提供键盘优先的多终端视图；两者观察和控制相同的后端 Agent，不重复启动进程。
- **Farming CRT 已成为可完整工作的界面**：包含实时终端缩略图、xterm session、New Agent、近期工作区、History、键盘导航、未读状态、终端字号、CRT 效果，以及返回 Farming Code 的入口。
- **文件工作流大幅完善，Review 基础能力更完整**：Project Files、Changes、Open Editors、搜索、预览、diff 和 blame 已形成产品流程；review snapshot、评论、偏好和 patchset state 具备项目级 API、UI state 与 demo 验证。
- **Agent runtime 更广、更真实**：Codex、Claude Code、OpenCode、Qoder、Qwen Code、Aider、GitHub Copilot CLI、Amazon Q、bash 和 zsh 按各自能力使用 provider-aware 启动参数、环境隔离和稳定元数据。
- **Codex runtime 选择更明确**：正常 CLI 仍是默认路径；实验性 App Server 为每个 Agent Home 隔离 runtime，并覆盖 transcript、请求、权限和恢复验证。
- **结构化 Chat 切换到 ACP**：Codex、Claude Code、OpenCode 和 Qoder 可以在保留 provider session 的情况下切换 Chat / Terminal，并支持原生权限、session 控件、图片 prompt 和后续消息排队。
- **前端或服务重启后 session 更可靠**：native PTY host、持久化 Farming session id、provider session 关联、终端屏幕恢复和重连行为都得到加固。
- **安装和更新路径具备发布语义**：npm 安装可使用应用内更新；源码 checkout 继续走 Git；独立 CLI / app bundle 包含 checksum 和 manifest。
- **移动端与终端交互经过系统打磨**：紧凑工作区、composer、终端尺寸与输入、中文输入法、链接、分享二维码，以及窄屏文件 / review 流程继续收敛。

## Farming CRT

- 新增独立 CRT 入口，采用参考复古终端的单色绿色视觉、中英文可读终端字体、轻量扫描效果和 GPU 友好的静态纹理层。
- 新增键盘优先的主页、New Agent、Settings 和 History；主页与 New Agent 选择器支持边界循环。
- 后端确认创建成功后，新 Agent 会自动成为当前键盘选择。
- 新增 History 自适应分页、provider 名称、受支持 Coding Agent 过滤、restore/resume/open 操作和稳定反色选择。
- 新增 10–20 px 的打开终端字号调节，拖动即时生效并自动持久化。
- 新增带 ANSI 颜色的终端缩略图、working 闪烁、未读外框、稳定卡片尺寸、可选 Dynamic Heat，以及更完整的项目 / 任务标题。
- 打开 session 默认统一使用 xterm.js；Ghostty Web 仅保留为显式调试开关。
- UI Theme 中新增 Farming Code 入口，可以在不改变后端 session 的情况下返回现代界面。
- 新增 ACP Chat 控件、后续消息排队、原生图片 prompt、`Alt+M` runtime 切换，以及中止后的 Composer 焦点恢复。
- 主页 Agent 网格稳定为预留的 `2 x 2`、`3 x 2` 和 `3 x 3` 机位；打开 New Agent 时会刷新可用 Agent，终端关闭快捷键更明确，并新增 Farming CRT 专属浏览器图标。

## Farming Code、Review 与 Files

- 扩展项目级 Files：搜索、Changes、Open Editors、文本编辑、Markdown 预览、diff、blame、创建 / 重命名 / 删除 / 移动和有界文件监听。
- 增强 review session API 和 UI state，覆盖不可变 revision、文件列表、patchset、评论、偏好、snapshot、隔离与 demo 验证。Review diff 新增字符级范围、文件末尾缺少换行提示，以及向上、向下或一次展开完整间隔的渐进式上下文控件。
- 改进 workspace 导航、近期工作区、路径建议、项目命名、文件 reveal、sticky context 和移动端侧边栏。延迟 reveal / focus 现在会尊重用户刚刚关闭的目录，也不会从文件右键菜单抢回焦点。
- 改进 Agent hover 详情、启动图标、深浅色图标变体、provider 分组、shell 标签和 runtime/profile 控件。
- 收敛 Codex transcript 清理、App Server 消息处理、goal 控制、权限切换，以及 terminal/chat fallback。ACP Chat 改为读取基于 revision 的 transcript 增量，以紧凑时间线展示进度更新和分组操作，大型 Tool 详情按需加载，并保留明确的中止状态。
- 新增基于 cursor 的 Agent session 分页、后端 provider 历史标题 / 项目搜索、History 页面内筛选、共用的 15 秒搜索超时，并稳定文件行内重命名的焦点与文件名主体选区。

## Runtime 与可靠性

- 交互 Agent 默认由独立 native PTY host 持有，支持服务重启恢复和无会话时的空闲清理。
- 新增稳定的 `fsess_*` Farming session id，并把 runtime id 与 provider 关联字段分开。
- 加固 shell 环境解析、login shell profile、prompt 归属、OSC busy/cwd 标记、终端状态、resize、屏幕重放、光标隔离和链接处理。New Agent 的发现与启动会读取最新 login-shell `PATH`，终端重放也会保留 TUI 使用的隐藏光标状态。
- 运行 History 只接收 Farming 明确支持的 Coding Agent；bash、zsh 和未知命令不会进入归档。
- 新增 Agent Home 元数据和 Codex App Server 的逐 Agent runtime 隔离。
- Agent turn 执行中会拒绝 runtime 切换；若目标 runtime 启动失败，Farming 会先恢复原 runtime 和 provider session，再明确报告失败。
- 改进更新回滚、安装类型识别、release manifest、checksum、包校验和 Linux 兼容打包。

## 发布与兼容性说明

- package、lockfile、release notes 和 Git tag 必须保持 `2.2.6` / `v2.2.6` 一致。
- 现有 `~/.farming/.session-token`、settings、sessions 和 history 会继续复用并自动规范化，无需手动迁移。
- 标准 App Bundle 使用目标系统的 Node.js 和 native runtime。独立的 Linux x64 legacy 包包含固定的 glibc 2.28 runtime，仅在宿主 glibc 低于 2.28 时启用；目标机仍需 Node.js 22。
- macOS CLI 产物采用 ad-hoc 签名，不是 Developer ID 签名，也未 notarize；部分系统首次运行可能需要手动允许一次。
- 本版本 GitHub Release 正文来自英文 release note 文件。

## 安装 / 升级

通过 npm 安装：

```bash
npm install -g farming-code@2.2.6
farming
```

也可以在 GitHub Release 下载对应产物：

| 平台 | CLI | App bundle |

| macOS Apple Silicon | `farming_2.2.6_darwin_arm64` | `farming-2.2.6-darwin-arm64.tar.gz` |

| Linux x64 | `farming_2.2.6_linux_amd64` | `farming-2.2.6-linux-x64.tar.gz` |

使用 `farming_2.2.6_checksums.txt` 和 `manifest.json` 校验下载产物。
